# samples/input

`sample_om.pdf` is a synthetic Offering Memorandum generated by the test fixture
builder (`tests/make_photo_fixture.py`). It is not a real broker document and is
safe to ship and re-run freely. It carries a synthetic cover (Offering Price
$4,250,000, NOI $297,500, Cap Rate 7.00%, 100 Example Way Springfield IL, tenant
Sample Tenant Co) and two embedded subject photos: one clean base stream under a
vector overlay (case 1) and one flattened raster with a contact bar, a diagonal
watermark, and a corner headshot (case 2).

## A note on the sample data output

The synthetic fixture was built to exercise the photo de-layering path, so its
cover text layer sits just under the 100-chars-per-page intake threshold. Running
`extract_one.py` on it therefore routes it to the human review queue and writes an
`extraction_status.json` rather than a `deal_data.json` (this is correct, expected
behavior for a thin-text OM). To still show the output shape, the shipped
`samples/output/deal_data.json` and `extraction_report.json` are illustrative:
generated by the real `shop1031_extract.schema` functions on the fixture's
synthetic cover values. The photo outputs in `samples/output/photos/` are produced
live by running `extract_photos.py` on the fixture.

## Bring your own OM

Drop any OM PDF in here and run:

```bash
python ../../scripts/extract_one.py "your_om.pdf" --out ../output
python ../../scripts/extract_photos.py "your_om.pdf" --out ../output/photos
```

A real OM with a normal text layer will produce a full `deal_data.json` instead of
routing to review. Use your own files for real extractions; the synthetic fixture
is only a shape example.
